# SmartQuery AI: Natural Language Analytics Platform 🚀

SmartQuery AI is a professional, serverless analytics workspace that allows users to perform complex data analysis on AWS-hosted datasets using only Natural Language. It acts as a "Semantic Layer" over your data lake, translating human questions into high-performance SQL.

## The Problem & Solution
**The Problem:** Non-technical business users often depend on data engineers to write SQL for simple insights, creating bottlenecks in decision-making.

**The Solution:** SmartQuery AI democratizes data by using **Llama 3.3** and **Amazon Athena** to provide an instant, conversational interface for any CSV or Excel dataset.

---

## Key Features

- **Automated Data Ingestion:** Upload raw CSV/Excel files; the system automatically cleans, converts, and pushes them to an **S3 Data Lake**.
- **Dynamic Data Catalog:** Automatically extracts schemas and manages metadata via **Spring Data JPA/Hibernate**, allowing the AI to understand any uploaded dataset instantly.
- **NL → SQL Engine:** Uses the **Llama 3.3 Versatile** model (via Groq API) to generate optimized Athena SQL, including advanced logic like `CAST` for mathematical operations on unstructured data.
- **Serverless Querying:** Leverages **Amazon Athena** for on-demand querying, ensuring high scalability and zero-cost maintenance.
- **Intelligent Visualization:** The UI automatically renders **Bar Charts** and provides human-readable **AI Insights** alongside the raw data grid.
- **SQL Audit Layer:** A transparent "Worksheet" view that allows technical users to audit the raw SQL generated by the AI.

---

## Technical Stack

- **Backend:** Java 17, Spring Boot 3.x, Spring Data JPA, Lombok.
- **Frontend:** React.js, Tailwind CSS, Recharts (Visualization), Lucide Icons.
- **Cloud (AWS):** S3 (Storage), Athena (Compute), Glue (Data Cataloging).
- **AI Layer:** Llama 3.3 (Groq Cloud API).
- **Database:** H2 (Local Metadata Storage).

---

## System Architecture

1.  **Ingest:** User uploads a file via the React UI.
2.  **Process:** Java Backend converts file to CSV and uploads it to **Amazon S3**.
3.  **Catalog:** System runs a `CREATE TABLE` command in **Athena** and stores metadata in a local **H2 DB**.
4.  **Query:** User asks: *"Show me total sales by region"* -> **Llama 3.3** generates the SQL.
5.  **Execute:** Java executes SQL via **AWS SDK** on the Athena engine.
6.  **Insights:** Results are fetched from S3, processed, and displayed as interactive tables and charts.

## Getting Started

### Prerequisites
- AWS Account (Free Tier)
- Groq Cloud API Key (Free Tier)
- Java 17+ and Node.js installed

### 1. Backend Setup
- Clone the repository.
- Navigate to `src/main/resources/` and rename `application.properties.example` to `application.properties`.
- Fill in your `AWS_ACCESS_KEY`, `AWS_SECRET_KEY`, and `AI_API_KEY`.
- Run the Spring Boot application on port `8082`.

### 2. Frontend Setup
- Navigate to the `smart-query-ui` folder.
- Run `npm install`.
- Run `npm run dev` to start the dashboard on `http://localhost:5173`.

## Security & Optimization
- **CORS Configuration:** Strictly controlled access for the React frontend.
- **SQL Sanitization:** Backend cleaning logic to strip markdown and prevent syntax errors in Athena.
- **Type Casting:** Automated `CAST(... AS DOUBLE)` logic to handle numerical analysis on string-based cloud datasets.
